5. KEY CONTENT POINTS

   [1]  The relocalisation pipeline refines an initial pose from the SLAM system
        proposed in \cref{chap:SLAM} using ICP against a 3DGS-rendered reference.

   [2]  Two independent sensor modalities are supported: stereo vision (ZED
        cameras) and dense LiDAR (Livox sensor).

   [3]  The 3DGS model is pre-built offline from SfM photogrammetry (described
        in \cref{subsec:system-context}). At registration time, it is rendered
        into point clouds at candidate viewpoints using GPU-accelerated
        rendering.

   [4]  FoundationStereo is used for stereo depth estimation. It produces dense
        disparity maps converted to coloured 3D point clouds. Configuration:
        downscale factor 0.7, z_far 1.0 m.

   [5]  Dual-camera mode merges front and back ZED point clouds via known
        extrinsic transforms before segmentation, increasing the angular
        coverage of the observation.

   [6]  The trunk segmentation pipeline (developed for this work) isolates vine
        trunk structure from noise using morphological closing, DBSCAN
        clustering, and voxel density filtering.

   [7]  The dense-mapper node (developed for this work) accumulates Livox LiDAR
        scans over a commanded interval to produce a dense point cloud.

   [8]  Patchwork++ removes the ground plane from the accumulated LiDAR cloud.

   [9]  The LiDAR processing pipeline (developed for this work) applies five
        sequential filtering stages: statistical outlier removal, angular
        filtering, radial outlier removal, normal-based filtering, and
        normal-based smoothing.

   [10] The ICP registration module (developed for this work) samples candidate
        viewpoints around the initial pose, renders the 3DGS model at each,
        runs ICP (Open3D tensor-based), and selects the best alignment.

   [11] LiDAR ICP uses a Cauchy robust kernel (k = 0.15) to mitigate outlier
        correspondences.

   [12] The TF correction system inserts a correction frame into the ROS 2 TF
        tree, projecting the sensor-frame ICP correction into the base_link
        frame without modifying the SLAM system's own transforms.

   [13] The pipeline follows a command pattern with asynchronous task queuing,
        thread-pool offloading, and semaphore-based GPU concurrency control.
